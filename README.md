# 🤖 OpenRouter Telegram Bot (Reborn)

**Мощный, быстрый и полностью асинхронный Telegram-бот для работы с нейросетями через OpenRouter.ai.**

Этот проект был переписан с нуля, чтобы предоставить максимальную производительность, удобство использования и легкость развертывания.

---

## ✨ Ключевые Возможности

### 🚀 Современный Стек
*   **Полностью асинхронный**: Построен на базе `python-telegram-bot` v20+ и `aiosqlite`. Никаких блокировок при работе с сетью или базой данных.
*   **Высокая производительность**: Способен обрабатывать множество запросов одновременно.

### 🧠 Умный Чат
*   **Стриминг ответов**: Бот печатает ответ по мере генерации (как ChatGPT), обновляя сообщение каждые 1.5 секунды.
*   **Поддержка контекста**: Бот помнит последние 20 сообщений диалога. Вы можете сбросить память кнопкой `🗑 Clear Context`.
*   **Smart Sort**: При выборе модели бот автоматически поднимает в топ самые надежные модели (Google, Meta, Mistral), затем популярные (DeepSeek), а экспериментальные убирает вниз.
*   **Пагинация**: В меню моделей добавлена кнопка `⬇️ Show More`, позволяющая просматривать весь список доступных моделей страницами.
*   **Роли**: Переключайтесь между режимами "Ассистент", "Переводчик", "Кодер" и другими.

### 🔑 Гибкость и Доступ
*   **Свои API ключи**: Используйте свой ключ OpenRouter для доступа к платным моделям.
*   **Поиск моделей**:
    *   **Авто-поиск**: Бот понимает названия моделей (напишите `gpt-4` или `claude` в чат).
    *   **Умная фильтрация**: Список бесплатных моделей отсортирован по надежности.
*   **Обработка ошибок**: Понятные сообщения при лимитах (Rate Limit 429) с кнопкой быстрой смены модели.

### 👮‍♂️ Администрирование
*   **Встроенная админка**: Команды для просмотра статистики (`/stats`) и управления пользователями.
*   **Баны**: Блокировка доступа (`/ban`, `/unban`).
*   **Логи ошибок**: Просмотр последних ошибок (`/logs`).

### 🐳 Docker-Ready
*   **Простой запуск**: Проект полностью докеризирован. Запуск одной командой `docker-compose up`.
*   **Изоляция**: База данных и логи хранятся в volume, код работает в изолированном контейнере.

### 🆕 Последние обновления (v2.2)
*   **Умный выбор моделей**:
    *   **Tier-Сортировка**: Надежные модели (Google, Meta) теперь всегда в топе.
    *   **Пагинация**: Возможность листать список моделей.
*   **Исправлена кодировка**: Полная поддержка UTF-8.
*   **Улучшенный UX**: Авто-сброс состояний и умный выход из поиска.

## 🛠 Установка и Запуск

Мы рекомендуем использовать Docker для запуска.

### Предварительные требования
*   Docker и Docker Compose
*   Токен бота от [@BotFather](https://t.me/BotFather)
*   API ключ от [OpenRouter](https://openrouter.ai/)
*   Ваш Telegram ID (узнайте у [@userinfobot](https://t.me/userinfobot))

### Шаг 1: Клонирование репозитория
```bash
git clone https://github.com/combx/openrouter-telegram-bot.git
cd openrouter-telegram-bot
```

### Шаг 2: Настройка окружения
Создайте файл `.env` на основе примера:
```bash
cp .env.example .env
nano .env
```
Заполните переменные:
*   `TELEGRAM_BOT_TOKEN`: Ваш токен бота.
*   `OPENROUTER_API_KEY`: Общий ключ для пользователей без своего ключа (обычно free-модели).
*   `ADMIN_ID`: Ваш ID администратора.

### Шаг 3: Запуск
```bash
docker-compose up -d --build
```
Бот запустится в фоновом режиме.

## 📖 Использование
После запуска найдите бота в Telegram и нажмите `/start`.

### Главное Меню
*   **🤖 Model**: Показывает текущую модель. Нажмите, чтобы найти и выбрать новую.
*   **🔑 My Key**: Меню управления вашим личным API ключом.
*   **👤 Profile**: Ваша статистика использования и настройки.
*   **🎭 Roles**: Выбор системной роли (промпта).

### Команды Админа
*   `/stats` - Показать количество пользователей и запросов.
*   `/ban <id>` - Заблокировать пользователя по ID.
*   `/unban <id>` - Разблокировать пользователя.
*   `/logs` - Показать последние записи из лога ошибок.

## 📂 Структура Проекта
*   `bot.py` - Точка входа в приложение.
*   `src/`
    *   `database/` - Модели SQLAlchemy и работа с БД.
    *   `handlers/` - Обработчики команд, сообщений и callback-кнопок.
    *   `services/` - Бизнес-логика (работа с API OpenRouter, управление пользователями).
    *   `utils/` - Вспомогательные утилиты (клавиатуры, форматтеры).
*   `Dockerfile` - Конфигурация образа Docker.
*   `docker-compose.yml` - Оркестрация сервисов.
